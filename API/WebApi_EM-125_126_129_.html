WEB API для приборов
EM-125, EM-126T, EM-129
Все запросы на устройство должны отправлятся методом 'GET /... HTTP/1.0'.
На запрос устройство возвращает JSON {...,"STATUS":"OK"} ответ, с вложенным результатом обработки запроса.
После входа на устройство (команда /api/login) в запрос необходимо включать полученный ключ сессии 'SID' (Пример: GET /73c637c6dbea14c75c/api/all/get?volt_rms).
ALL API's
Группа команд для работы с общими данными.
/api/all/get?sys_flag
Состояние флагов устройства.
Запрос:/api/all/get?sys_flag
Ответ:{"sys_flag":SYS_FLAGS,"STATUS":"OK"}
  • SYS_FLAGS - битовые флаги состояний на устройстве.
    • Бит 0 - идет отсчет времени АПВ по напряжению.
    • Бит 1 - идет отсчет времени АПВ по току.
    • Бит 2 - идет отсчет времени АПВ по мощности.
    • Бит 3 - идет отсчет времени задержки включения.
    • Бит 4 - состояние реле нагрузки (включено/отключено).
    • Бит 5 - зарезервирован.
    • Бит 6 - управление нагрузкой выполняется по событиям.
    • Бит 7 - управление нагрузкой выполняется пользователем вручную.
    • Бит 8 - управление нагрузкой заблокировано отпуском.
    • Бит 9 - время синхронизировано с сервером NTP.
    • Бит 10 - время синхронизировано с облаком my.overvis.com.
    • Бит 11 - лицевая панель заблокирована.
    • Бит 12 - идет счет времени АПВ по частоте.
    • Биты с 13 по 15 - зарезервированы.
    • Бит 16 - wi-fi в режиме клиента сети.
    • Бит 17 - wi-fi подключен и получен IP адрес.
    • Бит 18 - wi-fi в режиме точки доступа сети.
    • Бит 19 - wi-fi запущен и выдан IP адрес.
    • Биты с 20 по 25 - зарезервированы.
    • Бит 26 - WEB служба запущена.
    • Бит 27 - CLOUD служба запущена.
    • Бит 28 - NTP служба запущена.
    • Бит 29 - DNS служба запущена.
    • Биты с 30 по 31 - зарезервированы.
  • STATUS - результат обработки запроса.
/api/all/get?faul_flag
Состояние флагов ошибок устройства.
Запрос:/api/all/get?faul_flag
Ответ:{"faul_flag":FAUL_FLAGS,"STATUS":"OK"}
  • FAUL_FLAGS - битовые флаги ошибок на устройстве.
    • Бит 0 - верхний порог напряжения.
    • Бит 1 - нижний порог напряжения.
    • Бит 2 - обнаружено напряжение более 290 В.
    • Бит 3 - превышен лимит повторных включений по напряжению.
    • Бит 4 - порог по току.
    • Бит 5 - обнаружен ток более 17 А.
    • Бит 6 - превышен лимит повторных включений по току.
    • Бит 7 - порог по мощности.
    • Бит 8 - превышен лимит повторных включений по мощности.
    • Бит 9 - неисправность датчика температуры.
    • Бит 10 - обрыв датчика температуры.
    • Бит 11 - замыкание датчика температуры.
    • Бит 12 - сбой часов реального времени.
    • Бит 13 - неисправность реле нагрузки (наличие тока при отключенном реле).
    • Бит 14 - заблокирован по ограничению времени работы.
    • Бит 15 - верхний порог частоты.
    • Бит 16 - нижний порог частоты.
    • Бит 17 - превышен лимит повторных включений по частоте.
    • Биты с 18 по 29- зарезервированы.
    • Бит 30 - устройство не калибровано.
    • Бит 31 - повреждение настроек.
  • STATUS - результат обработки запроса.

  • user_info - имя устройства в base64;
  • STATUS - результат обработки запроса.

Получение соли для шифрования пароля.

Ответ:{"SALT":"7b962153","STATUS":"OK"}
  • SALT - соль;
  • STATUS - результат обработки запроса.

Вход на устройство, получение кода сессии.
Запрос:/api/login?login=PASSWODR
  • PASSWODR - пароль зашифрованный алгоритмом SHA1, PASSWORD = "EM-125" + "ПАРОЛЬ УСТРОЙСТВА" + SALT

  • SID - код сессии;
  • STATUS - результат обработки запроса. OK - успех, ERROR_LOGIN - ошибка.
